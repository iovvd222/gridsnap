# GridSnap

**グリッド解像度でウィンドウを自由に配置するクロスプラットフォームユーティリティ**

OS標準のスナップ機能（1/2・1/4分割）では粒度が粗すぎる。PowerToys FancyZones はゾーンを事前定義する方式で、組み合わせ爆発が避けられない。

GridSnap は **グリッド解像度という単一パラメータ** だけを定義し、各ウィンドウが実行時に何グリッド分占有するかを自由に決める。ゾーン定義の組み合わせ爆発を、グリッド解像度に圧縮する設計。

![GridSnap demo](assets/gridsnap.gif)

---

## 特徴

- **新規ウィンドウ自動配置** — ウィンドウ生成を検出し、事前に決定したグリッド上のセルへ自動配置。アプリ別ルール（`app_rules`）でデフォルト位置を宣言的に管理できる
- **Per-App キャプチャ** — ホットキー一発で「このアプリの現在位置を記憶」。設定に即永続化され、以降の自動配置に反映
- **リサイズスナップ** — ウィンドウ端ドラッグ後、ドロップ時に最寄りのグリッド線へ自動補正。`Shift+ドラッグ` でスナップ解除（微調整モード）
- **移動スナップ** — タイトルバードラッグ後、ドロップ時にグリッド交点へ自動補正。同じく `Shift+ドラッグ` で解除
- **アイドル時 CPU 0%・メモリ 5MB 以下** — 完全イベント駆動。ポーリングなし

## インストール

### Windows

1. [Releases](https://github.com/iovvd222/gridsnap/releases) から最新の `GridSnap-x.x.x.msi` をダウンロード
2. `.msi` を実行し、インストーラーの指示に従う
3. インストール完了後、GridSnap が自動的に起動しシステムトレイに常駐する

Windows ログオン時に自動起動するよう登録される。

### macOS

1. [Releases](https://github.com/iovvd222/gridsnap/releases) から最新の `GridSnap-x.x.x.dmg` をダウンロード
2. `.dmg` を開き、GridSnap を Applications フォルダにドラッグ
3. 初回起動時に「システム設定 → プライバシーとセキュリティ → アクセシビリティ」で権限を付与する

### ソースからビルド

```bash
git clone https://github.com/<your-username>/gridsnap.git
cd gridsnap
cargo build --release
```

ビルド要件: [Rust](https://www.rust-lang.org/tools/install) (edition 2021)

## 使い方

### 基本操作

1. GridSnap はインストール後、システムトレイに常駐する
2. ウィンドウをドラッグ → ドロップ時にグリッドへスナップ
3. `Shift` を押しながらドラッグ → スナップを無効化して自由配置

### グリッド設定

システムトレイのアイコンを右クリックし、グリッドの列数（Columns）・行数（Rows）を変更する。設定は即座に反映される。

### Per-App キャプチャ

1. ウィンドウを好きな位置に配置する
2. トレイメニューまたはホットキーから「このアプリの位置を記憶」を実行
3. 以降、そのアプリの新規ウィンドウは記憶した位置に自動配置される

### 一括移動

ホットキーで全可視ウィンドウを記憶済みルールに従って一括配置する。

## アーキテクチャ

```
┌──────────────────────────────────┐
│          GridSnap                │
│  ┌───────────┐ ┌──────────────┐  │
│  │ Key Hook  │ │ Config       │  │
│  │(Shift検出)│ │(グリッド定義) │  │
│  └─────┬─────┘ └──────┬───────┘  │
│        │               │         │
│  ┌─────▼───────────────▼──────┐  │
│  │   Snap Handler             │  │
│  │  (グリッドスナップ計算)      │  │
│  └─────────────┬──────────────┘  │
│                │                 │
│  ┌─────────────▼──────────────┐  │
│  │   Overlay Renderer         │  │
│  │  (グリッド線/タイトルバー)   │  │
│  └────────────────────────────┘  │
└──────────────────────────────────┘
```

- **Windows**: `SetWinEventHook` + `EVENT_SYSTEM_MOVESIZEEND` でドロップ後補正。DLL 注入不要
- **macOS**: `AXObserver` + `kAXWindowMovedNotification` / `kAXWindowResizedNotification` で同等の処理

## 既知の制約

- **管理者権限アプリ** — 対象が管理者権限で起動している場合、ウィンドウ操作が `ACCESS_DENIED` になる。GridSnap 側も管理者で起動すれば解決
- **一部 UWP アプリ** — Windows 標準アプリ（電卓等）はウィンドウ構造が特殊で、スナップが効かない場合がある
- **排他フルスクリーン** — ゲームの Exclusive Fullscreen モードは操作対象外
- **アプリ内部 UI** — タブバー・リボン等の内部レイアウト制御はスコープ外

## 技術スタック

- **言語:** Rust

- **macOS:** Accessibility API / CoreGraphics / Cocoa（`objc` + `cocoa` クレート）
- **設定:** TOML（内部形式。トレイ UI から操作）
